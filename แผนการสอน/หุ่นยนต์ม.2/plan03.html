แผนที่ 3 / 20 — Hr 5-6 (VEXcode VR)

แผนการจัดการเรียนรู้ที่ 3

หน่วยที่ 1 เรื่อง การเขียนโปรแกรมหุ่นยนต์เสมือน (VEXcode VR)จำนวน 6 ชั่วโมง
เรื่อง ภารกิจสรุปหน่วย (Disco / Castle Crasher)จำนวน 2 ชั่วโมง (ชั่วโมงที่ 5-6)
วิชา หุ่นยนต์กลุ่มสาระการเรียนรู้ วิทยาศาสตร์และเทคโนโลยี
ชั้นมัธยมศึกษาปีที่ 2ภาคเรียนที่ 1 ปีการศึกษา 2569
ผู้สอน นายภาณุพงศ์ อุ่นคำโรงเรียนบ้านป่ายุบ (บุ้นกีประชานุเคราะห์)
1. มาตรฐานการเรียนรู้/ตัวชี้วัด

มาตรฐาน ว 4.2 เข้าใจและใช้แนวคิดเชิงคำนวณในการแก้ปัญหา

ตัวชี้วัดระหว่างทาง

ว 4.2 ม.2/1 ออกแบบอัลกอริทึมที่ใช้แนวคิดเชิงคำนวณในการแก้ปัญหา

ว 4.2 ม.2/2 ออกแบบและเขียนโปรแกรมที่ใช้ตรรกะและฟังก์ชันในการแก้ปัญหา

2. จุดประสงค์การเรียนรู้
  1. รวมความรู้คำสั่ง Drivetrain, Loop, Sensor, If/Else ของหน่วยที่ 1 ได้ K
  2. เลือกและเขียนโปรแกรมแก้ภารกิจ Disco Dance หรือ Castle Crasher ได้สำเร็จ P
  3. วิเคราะห์ข้อแตกต่างระหว่าง Eye Sensor (ตรวจสี) และ Bumper Sensor (ตรวจการชน) K
  4. ประเมินผลงานของตนเองตาม Rubric ที่กำหนด P, A
  5. ภาคภูมิใจในผลงานหุ่นยนต์เสมือนที่สำเร็จและพร้อมเข้าสู่หน่วยถัดไป A
3. คุณลักษณะอันพึงประสงค์

☐ 1. รักชาติฯ ☐ 2. ซื่อสัตย์ ☑ 3. มีวินัย ☑ 4. ใฝ่เรียนรู้ ☐ 5. พอเพียง ☑ 6. มุ่งมั่น ☐ 7. รักไทย ☐ 8. จิตสาธารณะ

4. สมรรถนะสำคัญของผู้เรียน
  1. ความสามารถในการคิด (คิดสร้างสรรค์ + คิดเชิงตรรกะ)
  2. ความสามารถในการแก้ปัญหา
  3. ความสามารถในการสื่อสาร (นำเสนอผลงาน)
  4. ความสามารถในการใช้เทคโนโลยี
5. สาระสำคัญ

เมื่อรู้คำสั่งพื้นฐาน Drivetrain, Loop, Sensor และ If/Else ครบแล้ว ผู้เรียนสามารถสร้าง "โปรเจกต์" หุ่นยนต์ที่ทำงานเฉพาะทางได้ ภารกิจ Disco Dance ฝึกใช้ Eye Sensor ตรวจสีพื้น 4 สีและตอบสนองด้วยท่าทางที่ต่างกัน ภารกิจ Castle Crasher ใช้ Bumper Sensor + Distance Sensor หลบสิ่งกีดขวางพร้อมชนปราสาท เป็นการสังเคราะห์ความรู้ทั้งหมดที่เรียนมา

6. ความรู้เดิมที่นักเรียนต้องมี
  1. คำสั่ง Drivetrain, Loop, Sensor, If/Else จากแผนที่ 1-2
  2. การวิเคราะห์โจทย์และวางแผนการแก้ปัญหา
7. การจัดกิจกรรมการเรียนรู้
▶ ชั่วโมงที่ 5 (60 นาที) : เลือกภารกิจและออกแบบโปรแกรม

ขั้นที่ 1: การวิเคราะห์และกำหนดรายละเอียดของปัญหา (15 นาที)

  1. ครูเปิดเว็บไซต์ Robotm.2/Hr5-6VR.html ทบทวนสิ่งที่เรียนมา
  2. ครูแสดงตัวอย่างทั้ง 2 ภารกิจ:
    • ภารกิจ A: Disco Dance — Eye Sensor + If-else if + Down Eye
    • ภารกิจ B: Castle Crasher — Bumper + Distance + Drivetrain + Loop

    3. ภารกิจไหนน่าสนใจกว่าสำหรับนักเรียน? และทำไม?

    Disco สนุกเพราะหุ่นยนต์เต้นได้, Castle Crasher ท้าทายเพราะต้องคิดเรื่องการชน

ขั้นที่ 2: การวางแผนการแก้ปัญหา (45 นาที)

  1. นักเรียนเลือกภารกิจ A หรือ B ตามความสนใจ
  2. นักเรียนวางแผนตารางเงื่อนไข (Disco: สี → ท่าเต้น, Castle: เซ็นเซอร์ → การกระทำ)
  3. นักเรียนเขียน Pseudo code ลงในใบงานก่อนเริ่มเขียนโค้ดจริง
▶ ชั่วโมงที่ 6 (60 นาที) : เขียนโปรแกรมและประเมินสรุปหน่วย

ขั้นที่ 3: การดำเนินการแก้ปัญหา (35 นาที)

  1. นักเรียนเขียนโปรแกรมและทดสอบใน Playground ที่เลือก
  2. ครูเดินดูเป็นรายบุคคล แนะนำเทคนิคการ debug
  3. นักเรียนแคปวิดีโอหรือภาพหน้าจอแสดงผลลัพธ์

ขั้นที่ 4: การตรวจสอบและประเมินผล (25 นาที)

  1. นักเรียนทำใบงานที่ 3.1 "Final VEXcode VR Mission"
  2. นักเรียนทำแบบทดสอบสรุปหน่วย 1 (5 ข้อ) ในเว็บไซต์
  3. ครูสุ่มนักเรียน 3-5 คนนำเสนอผลงานหน้าชั้น

    4. เมื่อจบหน่วย 1 แล้ว นักเรียนคิดว่าหน่วย 2 (micro:bit) จะแตกต่างจาก VEXcode VR อย่างไร?

    micro:bit คือบอร์ดจริงจับได้ ไม่ใช่จำลองในคอม น่าจะต้องระวังเรื่องการเสียบสายและไม่ทำพัง

  4. ครูสรุปบทเรียนและประเมินผลตาม Rubric
8. ชิ้นงาน / ภาระงาน
  1. ใบงานที่ 3.1 "Final VEXcode VR Mission" + วิดีโอ ≥30 วินาที
  2. แบบทดสอบสรุปหน่วย (5 ข้อ)
