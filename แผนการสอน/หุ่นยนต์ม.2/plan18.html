แผนการจัดการเรียนรู้ที่ 18

หน่วยที่ 3 เรื่อง Cytron Maker Pi RP2040 + เว็บ Simulatorจำนวน 18 ชั่วโมง
เรื่อง โปรเจกต์จบ — ออกแบบ + เขียนโค้ด (Project Proposal)จำนวน 2 ชั่วโมง (ชั่วโมงที่ 35-36)
วิชา หุ่นยนต์กลุ่มสาระการเรียนรู้ วิทยาศาสตร์และเทคโนโลยี
ชั้นมัธยมศึกษาปีที่ 2ภาคเรียนที่ 1 ปีการศึกษา 2569
ผู้สอน นายภาณุพงศ์ อุ่นคำโรงเรียนบ้านป่ายุบ (บุ้นกีประชานุเคราะห์)
1. มาตรฐานการเรียนรู้/ตัวชี้วัด

มาตรฐาน ว 4.1 เข้าใจแนวคิด หลักการของเทคโนโลยีเพื่อการดำรงชีวิตในสังคมที่มีการเปลี่ยนแปลง

มาตรฐาน ว 4.2 เข้าใจและใช้แนวคิดเชิงคำนวณในการแก้ปัญหา

ตัวชี้วัดระหว่างทาง

ว 4.1 ม.2/2 ระบุปัญหาหรือความต้องการในชุมชนหรือท้องถิ่น สรุปกรอบของปัญหา

ว 4.1 ม.2/3 ออกแบบวิธีการแก้ปัญหา โดยวิเคราะห์เปรียบเทียบและตัดสินใจเลือกข้อมูล

ว 4.2 ม.2/2 ออกแบบและเขียนโปรแกรมที่ใช้ตรรกะและฟังก์ชันในการแก้ปัญหา

2. จุดประสงค์การเรียนรู้
  1. ระบุปัญหาในชีวิตจริงและออกแบบหุ่นยนต์เพื่อแก้ไขปัญหานั้นได้ K, P
  2. เขียน Planning Canvas (5 หัวข้อ) สำหรับโปรเจกต์ของตนเองได้ P
  3. เลือกโปรเจกต์จาก 6 ตัวเลือกที่กำหนด หรือเสนอไอเดียของตนเอง P, A
  4. เริ่มเขียนโค้ดส่วน SETUP ของโปรเจกต์ให้ทำงานได้ P
  5. เห็นคุณค่าของการวางแผนก่อนเขียนโค้ด A
3. คุณลักษณะอันพึงประสงค์

☐ 1. รักชาติฯ ☑ 2. ซื่อสัตย์ ☑ 3. มีวินัย ☑ 4. ใฝ่เรียนรู้ ☐ 5. พอเพียง ☑ 6. มุ่งมั่น ☐ 7. รักไทย ☑ 8. จิตสาธารณะ

4. สมรรถนะสำคัญของผู้เรียน
  1. ความสามารถในการคิด (คิดสร้างสรรค์ + วิเคราะห์)
  2. ความสามารถในการแก้ปัญหา
  3. ความสามารถในการสื่อสาร
  4. ความสามารถในการใช้เทคโนโลยี
5. สาระสำคัญ

โปรเจกต์จบเป็นการสังเคราะห์ความรู้ทั้ง 7 บทเรียนหน่วย 3 (NeoPixel, ปุ่ม, บัซเซอร์, Servo, DC Motor, HC-SR04, IR) ผสมผสานสร้าง "หุ่นยนต์ของตัวเอง" ที่แก้ปัญหาในชีวิตจริง ขั้นตอน Engineering Design: (1) Define ระบุปัญหา → (2) Plan วาง Planning Canvas → (3) Build เขียนโค้ดทีละส่วน → (4) Test ทดสอบและแก้ → (5) Present นำเสนอ มี 6 โปรเจกต์ตัวอย่าง: หุ่นยนต์ส่งของ, Line Racing, ประตูบ้านอัจฉริยะ, เปียโน+แสง, ไม้เท้านำทาง, Reaction Game ใช้อุปกรณ์ ≥3 ตัว + ตรรกะ if/while

6. ความรู้เดิมที่นักเรียนต้องมี
  1. ความรู้ทุกบทตั้งแต่แผน 11-17 (RP2040 ทุกอุปกรณ์)
  2. การวางแผนและเขียน Pseudo code
7. การจัดกิจกรรมการเรียนรู้
▶ ชั่วโมงที่ 35 (60 นาที) : เลือกโปรเจกต์และวางแผน

ขั้นที่ 1: การวิเคราะห์และกำหนดรายละเอียดของปัญหา (15 นาที)

  1. ครูเปิดเว็บไซต์ Robotm.2/Hr35-36RP.html
  2. ครูชี้แจงเงื่อนไขโปรเจกต์จบ:
    • ใช้อุปกรณ์ ≥3 ตัว
    • มีตรรกะ if/else หรือ while loop
    • ทำงานได้บนบอร์ดหรือ Simulator
    • นำเสนอใน Hr 39-40 (สอบไฟนอล)

    3. หุ่นยนต์ที่สามารถ "แก้ปัญหาในชีวิตจริง" ของนักเรียนได้ มีอะไรบ้าง?

    หุ่นยนต์รดน้ำต้นไม้ เครื่องนับคนเข้าห้อง เครื่องเตือนเมื่อมือถือใกล้สายตา ระบบล็อกประตูอัจฉริยะ

ขั้นที่ 2: การวางแผนการแก้ปัญหา (45 นาที)

  1. ครูแสดง 6 ตัวอย่างโปรเจกต์ในเว็บไซต์:
    • A: หุ่นยนต์ส่งของ (Motor + HC-SR04 + Servo + LED + บัซเซอร์)
    • B: Line Racing (Motor + IR ×3 + LED)
    • C: ประตูบ้านอัจฉริยะ (HC-SR04 + Servo + LED + บัซเซอร์)
    • D: เปียโน+แสง (ปุ่ม + บัซเซอร์ + NeoPixel + Servo)
    • E: ไม้เท้านำทาง (HC-SR04 + บัซเซอร์ + NeoPixel)
    • F: Reaction Game (NeoPixel + ปุ่ม + บัซเซอร์)
  2. นักเรียนเลือกโปรเจกต์ A-F หรือเสนอไอเดียของตนเอง (ครูพิจารณา)
  3. นักเรียนตอบ Planning Canvas 5 ข้อในใบงาน 18.1:
    1. ชื่อโปรเจกต์ + เป้าหมาย
    2. อุปกรณ์ที่ใช้ + ขา GPIO
    3. ตรรกะการทำงาน (ภาษาไทย)
    4. ฮาร์ดแวร์ที่ต้องเตรียม
    5. ความท้าทายที่คาดว่าจะเจอ
▶ ชั่วโมงที่ 36 (60 นาที) : เขียนโค้ดส่วน SETUP

ขั้นที่ 3: การดำเนินการแก้ปัญหา (45 นาที)

  1. ครูแสดง Template โค้ดเริ่มต้น (SETUP + FUNCTIONS + INITIAL + MAIN LOOP)
  2. นักเรียนเขียนโค้ดส่วน SETUP ของโปรเจกต์ตน (import + ตั้งค่าอุปกรณ์ทุกตัว)
  3. นักเรียนทดสอบ SETUP + LED + บัซเซอร์ ให้ทำงาน

ขั้นที่ 4: การตรวจสอบและประเมินผล (15 นาที)

  1. นักเรียนส่ง:
    • ไฟล์ planning.txt/pdf (5 คำถาม)
    • แคปโค้ด SETUP
    • แคป Sim ที่ LED + บัซเซอร์ทำงาน

    2. เหตุใดต้องวางแผน Planning Canvas ก่อนเขียนโค้ด?

    ลดเวลาเขียนใหม่ ป้องกันลืมอุปกรณ์ รู้ว่าจะแก้บั๊กตรงไหน เห็นภาพรวมก่อนเริ่ม

  2. ครูสรุปและประกาศว่าสัปดาห์หน้าทำต่อให้เสร็จ + ซ้อมนำเสนอ
8. ชิ้นงาน / ภาระงาน
  1. ใบงานที่ 18.1 "Project Proposal" — planning.txt + SETUP โค้ด + แคป Sim
