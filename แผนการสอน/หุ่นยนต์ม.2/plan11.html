5. สาระสำคัญ

RP2040 เป็นชิปประมวลผลออกแบบโดย Raspberry Pi Foundation ประเทศอังกฤษ มี CPU 2 cores 133 MHz, RAM 264 KB, GPIO 26 ขา บอร์ด Cytron Maker Pi RP2040 นำชิปนี้มาประกอบกับอุปกรณ์ในตัว: NeoPixel RGB ×2, ปุ่ม GP20/GP21, บัซเซอร์ Piezo (GP22), Motor Driver ×2, Servo Ports ×4, Grove Ports ×7 ทำให้สร้างหุ่นยนต์ได้ง่าย ใช้ภาษา CircuitPython เขียนโค้ดผ่าน Simulator บนเว็บก่อนลงบอร์ดจริง ลดความเสี่ยงเสียอุปกรณ์

6. ความรู้เดิมที่นักเรียนต้องมี
  1. ความรู้พื้นฐานเรื่องบอร์ดและเซ็นเซอร์จากหน่วย 2 micro:bit
  2. การคิดเป็นลำดับขั้นตอน
  3. การคัดลอกและพิมพ์ข้อความถูกต้องตามตัวพิมพ์
7. การจัดกิจกรรมการเรียนรู้
▶ ชั่วโมงที่ 21 (60 นาที) : ทำความรู้จัก RP2040 และ Maker Pi

ขั้นที่ 1: การวิเคราะห์และกำหนดรายละเอียดของปัญหา (15 นาที)

  1. ครูเปิดเว็บไซต์ Robotm.2/Hr21-22RP.html
  2. ครูแสดงผลคะแนนสอบกลางภาคและสรุปสิ่งที่เรียนใน 18 ชั่วโมงแรก
  3. ครูแนะนำเป้าหมายหน่วย 3: "หุ่นยนต์ของตัวเอง ที่นำเสนอท้ายภาคเรียน"

    4. หลังจากที่เคยใช้ micro:bit แล้ว นักเรียนคิดว่าจะนำมาทำหุ่นยนต์ที่เคลื่อนที่เองได้ไหม? ทำไมจึงต้องเปลี่ยนบอร์ด?

    น่าจะยากเพราะ micro:bit ไม่มี Motor Driver ในตัว ต้องใช้บอร์ดที่ขับมอเตอร์ได้ในตัวจะง่ายกว่า

ขั้นที่ 2: การวางแผนการแก้ปัญหา (25 นาที)

  1. ครูแนะนำชิป RP2040 (Raspberry Pi Foundation) — 2 cores, 133 MHz
  2. ครูแสดงรูป Cytron Maker Pi RP2040 พร้อมระบุอุปกรณ์ในตัวทุกชิ้น
  3. ครูแสดงตารางเปรียบเทียบ micro:bit vs RP2040 (CPU, GPIO, Motor, ภาษา)
  4. ครูประกาศใช้ภาษา CircuitPython (เลิกใช้ Block แล้ว!)

ขั้นที่ 3: การดำเนินการแก้ปัญหา (20 นาที)

  1. ครูพานักเรียนเข้าเว็บ Simulator (URL ที่ครูจะส่งให้)
  2. ครูสาธิตหน้าจอ Simulator: บอร์ดจำลอง (ซ้าย) + Code editor (ขวา)
  3. ครูเขียนโค้ดแรก: เปลี่ยน NeoPixel ดวงแรกเป็นสีแดง
▶ ชั่วโมงที่ 22 (60 นาที) : เขียน Python ครั้งแรกบน Simulator

ขั้นที่ 3 (ต่อ): การดำเนินการแก้ปัญหา (40 นาที)

  1. ครูอธิบายโครงสร้างโค้ด Python:
    • import board — นำเข้าโมดูล
    • import neopixel
    • pixels = neopixel.NeoPixel(board.GP18, 2)

  2. นักเรียนพิมพ์โค้ดและเปลี่ยนสีตามที่ตนเองชอบ
  3. นักเรียนทำใบงานที่ 11.1 + แคปหน้าจอ Simulator

ขั้นที่ 4: การตรวจสอบและประเมินผล (20 นาที)

  1. นักเรียนทำแบบทดสอบ 5 ข้อในเว็บไซต์

    2. ทำไม Python ถึงเข้มงวดเรื่องตัวพิมพ์ใหญ่-เล็ก ในขณะที่ MakeCode Block ไม่ต้องระวัง?

    เพราะ Python อ่าน "ข้อความ" ที่เราพิมพ์ตรงตัว board กับ Board เป็นคนละสิ่ง — Block ใช้กราฟิก ไม่มีปัญหานี้

  2. ครูสรุปบทเรียนและแจ้งว่าสัปดาห์หน้าจะเล่นกับ NeoPixel + บัซเซอร์
