แผนการจัดการเรียนรู้ที่ 2

หน่วยที่ 1 เรื่อง การเขียนโปรแกรมหุ่นยนต์เสมือน (VEXcode VR)จำนวน 6 ชั่วโมง
เรื่อง เซ็นเซอร์ + คำสั่งเงื่อนไข (If/Else) + ภารกิจ Wall Mazeจำนวน 2 ชั่วโมง (ชั่วโมงที่ 3-4)
วิชา หุ่นยนต์กลุ่มสาระการเรียนรู้ วิทยาศาสตร์และเทคโนโลยี
ชั้นมัธยมศึกษาปีที่ 2ภาคเรียนที่ 1 ปีการศึกษา 2569
ผู้สอน นายภาณุพงศ์ อุ่นคำโรงเรียนบ้านป่ายุบ (บุ้นกีประชานุเคราะห์)
1. มาตรฐานการเรียนรู้/ตัวชี้วัด

มาตรฐาน ว 4.2 เข้าใจและใช้แนวคิดเชิงคำนวณในการแก้ปัญหาที่พบในชีวิตจริงอย่างเป็นขั้นตอนและเป็นระบบ

ตัวชี้วัดระหว่างทาง

ว 4.2 ม.2/1 ออกแบบอัลกอริทึมที่ใช้แนวคิดเชิงคำนวณในการแก้ปัญหา หรือการทำงานที่พบในชีวิตจริง

ว 4.2 ม.2/2 ออกแบบและเขียนโปรแกรมที่ใช้ตรรกะและฟังก์ชันในการแก้ปัญหา

2. จุดประสงค์การเรียนรู้
  1. อธิบายหน้าที่ของเซ็นเซอร์ 5 ตัวใน VEXcode VR ได้ K
  2. อธิบายความแตกต่างระหว่าง if และ if-else พร้อมตัวอย่างได้ K
  3. เขียนโปรแกรมที่ใช้เซ็นเซอร์ + เงื่อนไข + Loop ร่วมกันได้ P
  4. ออกแบบอัลกอริทึมให้หุ่นยนต์ออกจากเขาวงกต (Wall Maze) ได้สำเร็จ P
  5. เห็นความสำคัญของการใช้เซ็นเซอร์เพื่อให้หุ่นยนต์ "ตัดสินใจ" ได้เอง A
3. คุณลักษณะอันพึงประสงค์

☐ 1. รักชาติฯ ☐ 2. ซื่อสัตย์ ☑ 3. มีวินัย ☑ 4. ใฝ่เรียนรู้ ☐ 5. พอเพียง ☑ 6. มุ่งมั่น ☐ 7. รักไทย ☐ 8. จิตสาธารณะ

4. สมรรถนะสำคัญของผู้เรียน
  1. ความสามารถในการคิด (การคิดเชิงตรรกะ การคิดเชิงนามธรรม)
  2. ความสามารถในการแก้ปัญหา
  3. ความสามารถในการใช้เทคโนโลยี
5. สาระสำคัญ

เซ็นเซอร์เป็นอุปกรณ์ที่ทำให้หุ่นยนต์ "รับรู้" สภาพแวดล้อม เช่น Distance Sensor วัดระยะ, Eye Sensor ตรวจสี, Bumper ตรวจการชน เมื่อรวมเข้ากับคำสั่งเงื่อนไข if/else จะทำให้หุ่นยนต์ ตัดสินใจได้เอง โดยไม่ต้องเขียนคำสั่งซ้ำๆ การใช้ forever ครอบโปรแกรมที่อ่านเซ็นเซอร์ + ตัดสินใจ คือหัวใจของหุ่นยนต์อัตโนมัติทุกประเภท

6. ความรู้เดิมที่นักเรียนต้องมี
  1. คำสั่ง Drivetrain และ Loop จากชั่วโมงที่ 1-2
  2. การเปรียบเทียบค่าทางคณิตศาสตร์ (<, >, =)
7. การจัดกิจกรรมการเรียนรู้
▶ ชั่วโมงที่ 3 (60 นาที) : รู้จักเซ็นเซอร์และเงื่อนไข

ขั้นที่ 1: การวิเคราะห์และกำหนดรายละเอียดของปัญหา (15 นาที)

  1. ครูเปิดเว็บไซต์ Robotm.2/Hr3-4VR.html และให้นักเรียนลงชื่อเข้าเรียน
  2. ครูตั้งคำถามนำเข้าสู่บทเรียน

    3. ถ้าหุ่นยนต์เดินตรงไปเรื่อย ๆ และเจอกำแพง มันจะรู้ได้อย่างไรว่าต้องเลี้ยว?

    ต้องมี "ตา" หรือเซ็นเซอร์ที่วัดระยะข้างหน้าได้ครับ/ค่ะ พอใกล้กำแพงก็ส่งสัญญาณให้โปรแกรมตัดสินใจเลี้ยว

ขั้นที่ 2: การวางแผนการแก้ปัญหา (25 นาที)

  1. ครูแนะนำเซ็นเซอร์ 5 ตัวใน VEXcode VR:
    • Distance Sensor — วัดระยะข้างหน้า (mm)
    • Front Eye Sensor — ตรวจสีที่อยู่ข้างหน้า
    • Down Eye Sensor — ตรวจสีที่พื้น
    • Bumper Sensor — ตรวจการชน (กด/ไม่กด)
    • Electromagnet — แม่เหล็กไฟฟ้า
  2. ครูแนะนำบล็อก if และ if-else พร้อมแสดงตัวอย่างเปรียบเทียบ:
if (Distance Sensor < 200 mm) { stop driving turn right 90 degrees } else { drive forward }

ขั้นที่ 3: การดำเนินการแก้ปัญหา (20 นาที)

  1. นักเรียนเปิด VEXcode VR เลือก Playground "Wall Maze"
  2. นักเรียนเขียนโปรแกรมที่ใช้ forever + if-else + Distance Sensor ตามตัวอย่างในเว็บไซต์
▶ ชั่วโมงที่ 4 (60 นาที) : ภารกิจตะลุย Wall Maze

ขั้นที่ 3 (ต่อ): การดำเนินการแก้ปัญหา (40 นาที)

  1. ครูแนะนำเทคนิค "Right-Hand Rule" เลี้ยวขวาเสมอเมื่อเจอกำแพง
  2. นักเรียนปรับโปรแกรมและทดสอบบน Wall Maze จนหุ่นยนต์ออกจากเขาวงกตได้สำเร็จ

    3. ถ้าตั้งระยะใน if เป็น 50 mm (ใกล้เกินไป) จะเกิดอะไรขึ้น?

    หุ่นยนต์จะชนกำแพงก่อนที่เซ็นเซอร์จะส่งสัญญาณให้เลี้ยวครับ/ค่ะ ควรตั้งระยะมากกว่านี้

  3. ครูแนะนำให้นักเรียนเลือกระยะปลอดภัย 150-200 mm

ขั้นที่ 4: การตรวจสอบและประเมินผล (20 นาที)

  1. นักเรียนทำใบงานที่ 2.1 "ตะลุย Wall Maze" + แคปวิดีโอหน้าจอ
  2. นักเรียนทำแบบทดสอบท้ายชั่วโมง 5 ข้อในเว็บไซต์
  3. ครูสุ่มนักเรียนนำเสนอวิธีคิดของตน + อภิปรายข้อจำกัดของหุ่นยนต์

    4. ทำไมหุ่นยนต์ที่ใช้แค่ Distance Sensor อาจไม่สามารถออกจากเขาวงกตที่ซับซ้อนมากได้?

    เพราะมันเห็นเฉพาะข้างหน้า ถ้าเขาวงกตมีทางแยกที่ต้องเลี้ยวซ้ายในบางจุด มันก็เลี้ยวขวาทุกครั้งตามที่เราเขียน อาจจะวนไม่จบ ต้องใช้ Bumper หรือเซ็นเซอร์ข้างเพิ่มครับ/ค่ะ

8. ชิ้นงาน / ภาระงาน
  1. ใบงานที่ 2.1 เรื่อง "ตะลุย Wall Maze" + วิดีโอแคปหน้าจอ
  2. แบบทดสอบท้ายชั่วโมง (5 ข้อ จากเว็บไซต์)

3. พฤติกรรมการทำงานสังเกตในชั้น4. แบบสังเกตพฤติกรรมระดับ "พอใช้" ผ่านเกณฑ์
