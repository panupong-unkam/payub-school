  4. เขียนโปรแกรม Line Follower ที่เลี้ยวนุ่มนวลตามตำแหน่งเส้น P
  5. เห็นคุณค่าของหุ่นยนต์ในการแข่งขัน (Line Following Race) A

5. สาระสำคัญ

IR Sensor (Infrared) ใช้แสงอินฟราเรดความถี่ที่ตาคนมองไม่เห็น (แต่กล้องมือถือเห็น) ส่ง LED IR + ตัวรับ Photodiode พื้นสีขาวสะท้อนแสง IR ดี → ตัวรับเห็นแสงเยอะ → ค่า 0 (LOW) พื้นสีดำดูดซับ → ตัวรับแทบไม่เห็น → ค่า 1 (HIGH) Maker Pi รองรับ IR Sensor 3 ตัว ต่อที่ขา GP4 (ซ้าย), GP5 (กลาง), GP6 (ขวา) เมื่อกลางเห็นเส้น (M=1) → เดินหน้า / ซ้ายเห็น (L=1) → เลี้ยวซ้ายนุ่ม / ขวาเห็น (R=1) → เลี้ยวขวานุ่ม / ทั้ง 3 ไม่เห็น (000) → หลุดเส้น → หยุด การ "เลี้ยวนุ่ม" ใช้ลดความเร็วล้อข้างหนึ่ง (เช่น drive(SPEED*0.3, SPEED)) แทนการหยุดล้อ

6. ความรู้เดิมที่นักเรียนต้องมี
  1. DC Motor + ฟังก์ชัน drive จากแผน 14
  2. เซ็นเซอร์ดิจิทัล (HIGH/LOW)
  3. การใช้เงื่อนไข if-elif-else

    2. รีโมตทีวีส่งสัญญาณอย่างไรไปที่ทีวี? ใช้แสงแบบไหน?

    ใช้แสงอินฟราเรดที่ตาคนมองไม่เห็น — ถ้าเปิดกล้องมือถือถ่ายจะเห็นแสงสีม่วงเรืองออกมาจากปลายรีโมต

ขั้นที่ 3: การดำเนินการแก้ปัญหา (20 นาที)

  1. นักเรียนต่อ IR Sensor 3 ตัวที่ GP4, GP5, GP6
  2. นักเรียนเขียนโค้ดอ่านค่า L, M, R + print() เพื่อตรวจสอบ
▶ ชั่วโมงที่ 34 (60 นาที) : Line Follower Race

ขั้นที่ 3 (ต่อ): การดำเนินการแก้ปัญหา (40 นาที)

  1. นักเรียนสร้างสนามทดสอบ: เทปกาวสีดำกว้าง 2-3 cm บนกระดาษ A3 ขาว ลากเป็นเส้นโค้ง
  2. นักเรียนเขียนโปรแกรม Line Follower:
    • M=1, L=R=0 → drive(SPEED, SPEED) เดินตรง
    • L=1, R=0 → drive(SPEED*0.3, SPEED) เลี้ยวซ้ายนุ่ม
    • R=1, L=0 → drive(SPEED, SPEED*0.3) เลี้ยวขวานุ่ม
    • L=M=R=0 → drive(0,0) + บัซเซอร์ดัง (หลุดเส้น)
  3. นักเรียนเพิ่ม NeoPixel แสดงสถานะ: เขียว=ตรง, น้ำเงิน=ซ้าย, ส้ม=ขวา
  4. นักเรียนทดสอบบนสนามจริง

ขั้นที่ 4: การตรวจสอบและประเมินผล (20 นาที)

  1. นักเรียนทำใบงานที่ 17.1 + วิดีโอ + ภาพสนาม
  2. นักเรียนทำแบบทดสอบ 5 ข้อ

    3. เหตุใดต้อง time.sleep(0.02) สั้นๆ ใน Line Follower?

    เพื่อให้เซ็นเซอร์อ่านค่าถี่ๆ ทันการเลี้ยว ถ้านานเกินไป หุ่นยนต์จะออกนอกเส้นก่อนตอบสนอง

  3. ครูสรุปบทเรียนและประกาศว่าสัปดาห์หน้าเริ่มโปรเจกต์จบ!
