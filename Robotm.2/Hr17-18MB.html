📡 Radio บน micro:bit

ใช้คลื่นวิทยุ 2.4 GHz ส่ง-รับข้อมูลระหว่าง micro:bit หลายบอร์ดในระยะ ~70 เมตร

  • ส่งได้: ตัวเลข, ข้อความ, คู่ key-value
  • ไม่ต้องใช้: WiFi, Bluetooth ต่อมือถือ
  • ทำงานเป็นกลุ่ม — บอร์ดที่อยู่ "กลุ่มเดียวกัน" จะคุยกันได้

📤 ด่านที่ 2: บล็อก Radio พื้นฐาน

ใช้ 3 บล็อกหลักนี้ ทำได้ทุกอย่าง

🔹 กำหนดกลุ่ม

on start: radio set group 1

ตั้งกลุ่ม 0-255 — บอร์ดที่กลุ่มเดียวกันจะคุยกันได้

🔹 ส่ง (Send)

on button A pressed: radio send number 100 radio send string "HI"

🔹 รับ (Receive) — ทำงานเมื่อมีข้อความเข้ามา

on radio received receivedNumber: show number receivedNumber on radio received receivedString: show string receivedString
⚠️ สำคัญ: ถ้าบอร์ดอยู่กลุ่ม "0" ทุกบอร์ดในห้องจะคุยกันมั่ว — ตั้ง กลุ่มเฉพาะคู่ของตัวเอง (เช่น 1, 2, 3)!

💬 ด่านที่ 3: ตัวอย่าง — Chat ระหว่างเพื่อน

โปรแกรมเดียว → flash ลงบอร์ดทั้งของฉันและของเพื่อน → กดปุ่มคุยกันได้!

📚 โค้ดเต็ม (ใช้ทั้ง 2 บอร์ด)

on start: radio set group 7 // กลุ่มของคู่เรา on button A pressed: radio send string "HI" on button B pressed: radio send string "BYE" on radio received receivedString: show string receivedString pause 1000 clear screen
💡 ฝึกคิด:

โค้ดเดียวกัน — แต่ละบอร์ดทั้ง ส่งและรับ ได้พร้อมกัน! ไม่ต้องเขียนแยกฝั่ง

🎮 ด่านที่ 4: ตัวอย่างขั้นสูง — เป่ายิ้งฉุบไร้สาย

2 บอร์ดเล่นเป่ายิ้งฉุบกัน — เขย่าเลือก → ส่งให้อีกฝั่ง → ดูผล

📚 โค้ดเป่ายิ้งฉุบ

on start: radio set group 5 set myChoice to 0 on shake: // เขย่าเลือก set myChoice to pick random 1 to 3 // 1=ค้อน 2=กระดาษ 3=กรรไกร if myChoice == 1: show icon Rectangle // ค้อน else if myChoice == 2: show icon SmallSquare // กระดาษ else: show icon Scissors // กรรไกร radio send number myChoice on radio received receivedNumber: // ได้รับของเพื่อน pause 1500 if receivedNumber == myChoice: show string "DRAW" else if (myChoice == 1 and receivedNumber == 3) or (myChoice == 2 and receivedNumber == 1) or (myChoice == 3 and receivedNumber == 2): show icon Happy // ชนะ else: show icon Sad // แพ้
🎯 ทำไมต้อง pause 1500?

เพื่อให้ผู้เล่นเห็นว่าตัวเองเลือกอะไรก่อน แล้วถึงแสดงผลลัพธ์ — ไม่งั้นจะเปลี่ยนเร็วเกินไป

✅ เกณฑ์ประเมิน (เต็ม 10 คะแนน):
  1. ใช้เซ็นเซอร์/ฟีเจอร์อย่างน้อย 2 ตัว รวมกัน (2 คะแนน)
  2. มี เงื่อนไข if-else ที่สมเหตุสมผล (2 คะแนน)

  5. ความสมบูรณ์ + ความคิดสร้างสรรค์ (2 คะแนน)
